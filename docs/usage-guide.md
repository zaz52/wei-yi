# 唯一使用说明

本文档用于放入“唯一”的飞书使用说明页，适合新用户按步骤完成安装、模型配置、项目创建、大纲整理、章节写作和记忆维护。

## 1. 唯一是什么

唯一是一款面向长篇小说创作的 AI 写作桌面软件。它不是单纯的聊天窗口，而是围绕小说项目组织大纲、章节、人物、设定、伏笔、时间线、审查结果和记忆资料。

它的核心目标是解决长篇 AI 写作中常见的几个问题：

- 写到后期 AI 遗忘前文
- 人物性格和关系前后不一致
- 世界观、地点、组织、道具设定混乱
- 伏笔丢失或回收不完整
- 章节节奏偏离大纲
- 需要反复翻旧章节找资料

推荐使用顺序：

1. 创建或打开小说项目
2. 配置可用的大模型
3. 写入故事大纲、卷大纲、角色设定和世界观资料
4. 提取初始记忆
5. 生成或手写章节
6. 保存正式章节并沉淀章节记忆
7. 使用审查、改写、图谱和搜索功能维护长期一致性

## 2. 第一次使用流程

第一次使用不要急着直接让 AI 写正文。更稳定的流程是先让软件理解你的小说资料。

### 2.1 打开或创建项目

项目可以理解为一个小说文件夹。项目中会保存：

- `wiki/chapters`：正式章节
- `wiki/outlines`：大纲和细纲
- `wiki/entities`：人物、地点、组织、物品等实体记忆
- `wiki/memory`：章节记忆、伏笔、时间线、角色状态等结构化资料
- 资料库和检索数据

建议每本小说单独建一个项目，不要把多本小说混在一个项目里。

### 2.2 配置模型

进入“设置 > 模型设置”，填写你的模型服务信息。

常见配置项包括：

- 接口地址：模型服务的 API 地址
- API Key：模型服务密钥
- 模型名称：实际调用的模型 ID
- API 模式：OpenAI 兼容、Responses API 或 Anthropic 兼容

配置后先点击测试模型。测试通过后再进入写作流程。

### 2.3 准备大纲和基础资料

AI 写作质量高度依赖你给它的上下文。建议先准备：

- 故事简介
- 世界观设定
- 主线剧情
- 分卷大纲
- 角色小传
- 关键地点和组织
- 重要物品、能力体系和规则
- 已知伏笔和禁写规则

资料越清晰，后续章节越不容易跑偏。

### 2.4 提取初始记忆

当大纲和基础资料准备好后，使用“提取初始记忆”。这一步会把你的大纲内容整理成软件可检索、可追踪的结构化记忆。

提取初始记忆后，AI 在写章节时能更容易获得：

- 主要人物状态
- 故事阶段目标
- 当前世界观规则
- 已建立的人物关系
- 待推进的伏笔
- 不能违反的正史设定

## 3. 模型设置建议

模型决定生成质量、速度和稳定性。

### 3.1 LLM 大模型

LLM 负责正文生成、改写、总结、审查和推理。建议使用上下文长度较高、中文写作能力稳定的模型。

如果模型经常出现以下问题，建议更换模型或降低单次任务复杂度：

- 输出明显跑题
- 不按格式返回
- 忘记大纲要求
- 长章节中后半段质量下降
- 角色语气频繁变化

### 3.2 向量嵌入模型

向量嵌入用于语义检索。它能帮助软件找到和当前章节相关的旧设定、旧事件和角色资料。

如果不配置嵌入模型，部分语义搜索能力会下降，但基础写作功能仍可使用。

### 3.3 多模态模型

如果你需要导入含图片的资料，或者让软件理解图片内容，需要配置支持视觉输入的模型。纯文本模型无法处理图片，会在图片解析时失败。

## 4. 大纲怎么写

大纲是小说写作的骨架。唯一支持把大纲拆成不同层级，方便长篇项目维护。

### 4.1 故事总纲

总纲用于说明整本小说的核心方向：

- 主角是谁
- 主角想要什么
- 最大阻力是什么
- 故事最终会走向哪里
- 主要爽点、冲突点和情绪线是什么

### 4.2 分卷大纲

分卷大纲用于控制阶段目标。每一卷建议写清楚：

- 本卷开局状态
- 本卷核心矛盾
- 主要敌人或阻碍
- 关键事件节点
- 本卷结尾变化

### 4.3 章节细纲

章节细纲用于指导单章写作。建议包含：

- 本章目标
- 出场人物
- 发生地点
- 冲突推进
- 情绪变化
- 结尾钩子
- 必须出现或不能出现的内容

章节细纲越具体，AI 生成正文时越稳定。

## 5. 章节写作流程

推荐的正式写作流程如下：

1. 确认当前章节细纲
2. 检查上一章结尾和当前人物状态
3. 生成上下文
4. 生成章节草稿
5. 人工阅读并修改
6. 使用审查功能检查问题
7. 必要时使用 AI 改写
8. 保存为正式章节
9. 让软件沉淀章节记忆

### 5.1 生成章节前检查什么

写新章节前建议确认：

- 当前章是否有明确目标
- 上一章结尾是否能自然衔接
- 角色当前状态是否正确
- 本章是否需要推进某条伏笔
- 本章是否会改变人物关系或势力局面

### 5.2 正式章节保存后会发生什么

保存正式章节后，软件会尝试从正文中提取结构化记忆，包括：

- 章节摘要
- 出场人物
- 关键地点
- 组织、物品、能力和规则
- 事件变化
- 角色状态变化
- 人物关系变化
- 新增、推进或回收的伏笔
- 时间线信息

这些记忆会在后续章节写作中被检索和复用。

## 6. 审查和改写

审查功能用于检查章节是否存在问题。常见审查方向包括：

- 剧情是否偏离大纲
- 人物性格是否崩坏
- 设定是否前后矛盾
- 节奏是否拖沓
- 伏笔是否遗漏
- 上下文衔接是否自然

AI 改写适合处理局部问题。建议不要无脑接受全部改写，尤其是重要章节，应当逐条检查。

## 7. 资料库和搜索

资料库用于存放参考资料、设定资料和导入内容。软件会尽量把资料整理成可检索的页面。

搜索适合用于：

- 查找某个角色之前出现过什么
- 查找某个设定最早在哪里建立
- 查找伏笔是否已经回收
- 查找某个地点、组织、物品的相关资料
- 写新章节前补上下文

## 8. 图谱和记忆中心

图谱用于查看人物、地点、组织、事件之间的关系。长篇小说越复杂，图谱越有价值。

记忆中心用于管理长期信息，包括：

- 人物状态
- 关系变化
- 伏笔状态
- 时间线
- 正史设定
- 章节摘要

如果 AI 写出来的内容和前文冲突，优先检查记忆中心里相关资料是否缺失、过旧或互相矛盾。

## 9. 常见问题

### 9.1 为什么 AI 不按我的大纲写

常见原因：

- 大纲太短，缺少关键约束
- 模型能力不足
- 当前章节任务太复杂
- 没有提取初始记忆
- 上下文里存在互相冲突的旧资料

处理办法：

- 把章节目标写得更具体
- 拆短单次生成任务
- 检查模型配置
- 重新整理相关大纲和记忆

### 9.2 为什么人物越写越不像

通常是角色资料不够稳定，或者新章节没有带到角色状态。建议为主要角色维护：

- 人物小传
- 性格底层逻辑
- 说话习惯
- 当前目标
- 当前关系
- 最近重大经历
- 不能违反的行为边界

### 9.3 为什么保存章节后记忆不完整

可能原因包括：

- 模型没有按要求返回结构化内容
- 章节内容过长导致信息被裁剪
- 章节里事件表达不清楚
- 模型能力不足

可以手动补充重要人物、伏笔和设定，或者将特别重要的内容写入正史设定。

### 9.4 为什么有些模型不能用

常见原因：

- API 地址填写错误
- API Key 无效
- 模型名称和服务商实际名称不一致
- 选择了不支持当前 API 模式的模型
- 当前模型不支持图片或长上下文

先在设置里测试模型，测试通过后再写章节。

## 10. 推荐使用习惯

- 每一章都先写章节目标
- 重要角色都建立小传
- 重要规则写进正史设定
- 每次大改剧情后重新检查记忆
- 不要把所有内容塞进一次生成任务
- AI 生成后必须人工读一遍
- 章节正式保存前先审查
- 长篇连载要定期检查人物关系、伏笔和时间线

## 11. 最短上手流程

如果只想快速开始，可以按这个流程：

1. 新建小说项目
2. 在设置里配置并测试模型
3. 写一份故事总纲
4. 写第一卷大纲
5. 写第一章细纲
6. 提取初始记忆
7. 生成第一章草稿
8. 人工修改
9. 保存正式章节
10. 继续下一章

先把这个闭环跑通，再逐步使用图谱、审查、改写、资料库和记忆中心。
